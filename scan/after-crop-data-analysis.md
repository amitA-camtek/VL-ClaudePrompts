# "after-crop" Folder - Data Relationship Analysis

## Overview

The folder contains **5,220 JPEG image tiles** generated by cropping **38 unique scans** into a grid of smaller tiles for downstream processing (e.g., defect detection AI).

---

## Filename Convention

Each filename follows this pattern:

```
{ID1}.{ID2}.c.{HASH}.{VER}_x{X}_y{Y}_slot{SLOT}.jpeg
```

### Breakdown

| Field   | Example             | Meaning                                         |
|---------|---------------------|-------------------------------------------------|
| ID1     | `105639`            | Primary identifier (likely wafer/lot/scan ID)   |
| ID2     | `33124`             | Secondary identifier (likely die/site index)    |
| `c`     | `c`                 | Constant literal - possibly "crop" or "channel" |
| HASH    | `1482081634`        | Signed integer hash (can be negative)           |
| VER     | `2`                 | Version / revision number (1 or 2)              |
| X       | `618`               | X pixel offset of this tile in the source image |
| Y       | `412`               | Y pixel offset of this tile in the source image |
| SLOT    | `10`                | Slot number (constant = `10` for all files)     |

**Full example:** `105639.33124.c.1482081634.2_x618_y412_slot10.jpeg`

## Visual Content

The tiles are **microscopic inspection images** of semiconductor/electronic components. They show:
- Dark circular/rectangular structures (likely die pads, vias, or solder bumps)
- Yellow/golden patterned background (likely PCB traces or wafer circuitry)
- Cracking/branching patterns in the substrate
